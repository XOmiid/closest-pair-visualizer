📖 Overview: What is this problem? ▼

The Closest Pair of Points problem asks: given n points in a plane, find the two points with the smallest Euclidean distance between them.

Why it matters: Used in collision detection, clustering, geographic information systems, computer graphics, and molecular modeling.

High-level idea (D&C): Split points by median x-coordinate. Recursively find the closest pair in each half. Then check points near the dividing line (the "strip") for possible closer pairs.

🧮 Key Concepts ▼

Divide & Conquer: Split → Solve sub-problems → Combine. The key insight is the strip optimization.

Strip optimization: After finding δ = min(d_left, d_right), we only check points within distance δ of the midline. Within the strip, for each point you only need to check at most 7 neighbors sorted by y-coordinate. This is because at most 8 points can fit in a 2δ × δ rectangle.

Common mistakes:

  • Forgetting the strip check entirely
  • Checking too many strip neighbors (more than 7)
  • Not sorting by x before dividing
  • Using brute force in the strip (defeats the purpose)
📝 Worked Example ▼

Step 1: Sort points by x-coordinate: (1,1), (2,3), (4,2), (5,5), (7,1), (9,4)

Step 2: Divide at median: Left = {(1,1),(2,3),(4,2)}, Right = {(5,5),(7,1),(9,4)}

Step 3: Recursively solve: dL = dist((1,1),(2,3)) = 2.24, dR = dist((7,1),(9,4)) = 3.61

Step 4: δ = min(2.24, 3.61) = 2.24. Build strip: all points within x ∈ [4-2.24, 4+2.24]

Step 5: Check strip pairs. The final answer is the minimum of dL, dR, and any strip improvement.

CLO-2: Sorting & Strip Searching

The D&C algorithm relies on two critical sorting steps:

1. Sort by x-coordinate — enables the divide step. Without this, we cannot find the median and split points correctly.

2. Sort strip by y-coordinate — enables efficient strip checking. By sorting by y, each point only checks the next 7 neighbors (points within δ in y-direction).

What if sorting is removed? We'd need to check all points in the strip against each other — O(n²) in the worst case, defeating the purpose of D&C entirely. The algorithm degrades to O(n² log n).

THE 7-COMPARISON RULE

In the strip of width 2δ, sort points by y-coordinate. For each point, check only the next 7 points above it. Why 7? In a 2δ × δ rectangle, you can pack at most 8 points (4 in each δ × δ half) while maintaining all pairwise distances ≥ δ. So you never need more than 7 comparisons per point. This keeps the strip check at O(n).

Without x-sort: Cannot divide correctly → O(n²) or worse

Without y-sort in strip: Must compare all strip pairs → O(n²) strip check

With both sorts: Strip check = O(n), total = O(n log n) ✓
